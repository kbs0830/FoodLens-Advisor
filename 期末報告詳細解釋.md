# FoodLens-Advisor — 期末報告（詳細解釋）

## 一、專題摘要
本專題實作一個跨平台的「FoodLens-Advisor」系統，結合前端 YOLO 物件檢測、Web 前端展示、以及以 FastAPI 建立的後端 BFF（Backend-for-Frontend）服務，提供圖像與文字（YOLO 偵測結果）之飲食分析與營養建議。此專案以模組化設計（前端、BFF、啟動腳本）為主，方便分工與擴充。

## 二、專案目標
- 建立可本地啟動的開發環境（含前端靜態伺服器與 FastAPI BFF）。
- 支援圖像（Base64）與文字（YOLO 檢測清單）兩種分析請求。
- 提供可替換的 AI/視覺供應商（mock / 真實 API）以便測試與擴充。

## 四、重要檔案詳細說明
- `start-all.bat`：一鍵啟動 BFF（uvicorn）與前端（Python `http.server`）。改進要點：
  - 設定 `chcp 65001` 為 UTF-8，避免中文日誌亂碼；
  - 以順序檢查並優先使用 `foodlens-web/bff-fastapi/.venv`、repo `.venv`，最後 fallback 到系統 `python`；
  - 檢查 8080/8000 佔用並嘗試 `taskkill`（日誌寫入 `logs`）；
  - 以新視窗（cmd start）啟動服務，保持原始視窗可用於檢視啟動狀態。

- `foodlens-web/bff-fastapi/requirements.txt`：列出 BFF 需要的套件（例如 `fastapi`、`uvicorn`、`pydantic`、`python-dotenv`）。

- `foodlens-web/bff-fastapi/app/main.py`：FastAPI 應用入口，定義路由：
  - `GET /health`：回傳狀態檢查。
  - `POST /api/v1/analyze-food`：接受 `AnalyzeFoodRequest`（含 `image_base64`）並回傳 `AnalyzeFoodResponse`。
  - `POST /api/v1/analyze-text`：接受 `AnalyzeTextRequest`（YOLO 偵測文字）並回傳分析結果。

- `foodlens-web/bff-fastapi/app/schemas.py`：Pydantic 定義資料模型（請求/回應與內嵌模型 `MacroNutrients`、`DietaryRuleCheck`）。

- `foodlens-web/bff-fastapi/app/services/vision_service.py`：核心分析邏輯（目前支援 `mock` provider）：
  - `analyze_with_provider()`：處理圖像分析的呼叫，mock 回傳示範結果或回退訊息；
  - `analyze_text_with_ai()`：處理 YOLO 檢測文字的分析（mock 回應）；
  - `_create_analysis_response()`：使用簡易 nutrition DB（字串比對）計算總熱量與巨量營養素，並檢查飲食規則。

## 五、環境需求與安裝
- 建議使用 Python 3.11/3.10 或相容版本。
- 建議在 `foodlens-web/bff-fastapi` 使用虛擬環境安裝相依：

```powershell
cd foodlens-web\bff-fastapi
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt
```

## 六、啟動與測試（詳細步驟）
1. 啟動整套系統（BFF + 前端）:

```powershell
cd "C:\Users\ASUS\Desktop\人工智慧與深度學習應用期末專題"
cmd /c start-all.bat
```

2. 單獨啟動 BFF（開發時常用）:

```powershell
cd foodlens-web\bff-fastapi
.\.venv\Scripts\python.exe -m uvicorn app.main:app --host 0.0.0.0 --port 8080
```

3. 驗證 health endpoint:

```powershell
Invoke-RestMethod -Uri http://127.0.0.1:8080/health
# -> {"status":"ok"}
```

4. 呼叫 API 範例（分析文字）：

```powershell
$body = @{
  food_items = @("chicken breast","broccoli")
  description = "grilled chicken breast with steamed broccoli"
} | ConvertTo-Json
Invoke-RestMethod -Uri http://127.0.0.1:8080/api/v1/analyze-text -Method POST -Body $body -ContentType 'application/json'
```

範例回應（mock）:

```json
{
  "food_items": ["chicken breast","broccoli"],
  "estimated_calories_kcal": 312.0,
  "macros": {"protein_g": 48.0, "carbs_g": 8.0, "fat_g": 9.0},
  "rule_check": {"high_protein": true, "zero_starch": false, "zero_alcohol": true, "mild_not_spicy": true},
  "next_meal_suggestion": "下一餐可增加葉菜類與水分，維持蛋白質攝取。"
}
```

## 七、日誌與偵錯
- 啟動腳本與服務日誌位於專案 `logs/` 資料夾：
  - `bff.log`：BFF 啟動與執行日誌（UTF-8）。
  - `web.log`：前端 http.server 日誌。
  - `bff_stop.log` / `web_stop.log`：`taskkill` 嘗試時的輸出。

- 常見問題處理：
  - 埠被佔用（10048）：檢查 `netstat -aon | findstr ":8080"`，用 `taskkill /PID <pid> /F` 終止或重啟電腦。
  - Python 找不到：確認 `.venv\Scripts\python.exe` 是否存在，或安裝系統 python 並加入 PATH。

## 八、測試策略
- 單元測試：將來可為 `vision_service` 的 `_create_analysis_response` 撰寫 pytest 測試，覆蓋多種食物組合與邊界值。
- 集成測試：啟動 BFF 與前端後，使用 HTTP 測試套件（pytest + httpx）自動化呼叫 API 並驗證回傳格式。

# FoodLens-Advisor — 期末報告（詳細解釋）

## 一、專題摘要
本專題實作一個跨平台的「FoodLens-Advisor」系統，結合前端 YOLO 物件檢測、Web 前端展示、以及以 FastAPI 建立的後端 BFF（Backend-for-Frontend）服務，提供圖像與文字（YOLO 偵測結果）之飲食分析與營養建議。

## 二、目標
- 建立可本地啟動的開發環境（含前端靜態伺服器與 FastAPI BFF）。
- 支援圖像（Base64）與文字（YOLO 檢測清單）兩種分析請求。
- 提供可替換的 AI/視覺供應商（mock / 真實 API）以便測試與擴充。

## 三、系統架構概覽
- Android client（選配）: 提供行動端上傳與 UI。
- Web client: 提供 YOLO 前端檢測與展示（位於 `foodlens-web/web-client`）。
- BFF (FastAPI): 位於 `foodlens-web/bff-fastapi/app`，負責接收前端請求、呼叫視覺/AI 服務並回傳結果。
- 啟動腳本: `start-all.bat`（改進版已加入 UTF-8、venv 偵測與自動釋放被佔用埠的邏輯）。

## 四、重要檔案說明
- `start-all.bat`：一鍵啟動 BFF（uvicorn）與前端（Python http.server），若埠被佔用會嘗試釋放。已改善：
  - 使用 `chcp 65001` 設定 UTF-8 控制台編碼，避免中文日誌亂碼。
  - 先檢查並優先使用 `bff/.venv` 的 Python，再 fallback 到 repo venv 或系統 Python。
  - 對 `taskkill` 操作加上日誌備援。

- `foodlens-web/bff-fastapi/app/main.py`：FastAPI 應用入口，含 `/health`、`/api/v1/analyze-food` 與 `/api/v1/analyze-text`。
- `foodlens-web/bff-fastapi/app/schemas.py`：使用 Pydantic 定義請求與回應資料結構。
- `foodlens-web/bff-fastapi/app/services/vision_service.py`：提供 mock 分析與簡單 nutrition DB，供測試與開發使用。

## 五、實作重點與技術細節
- BFF 使用 FastAPI + Uvicorn；資料驗證由 Pydantic 管理。
- 設計 `analyze_text` 端點以接收 YOLO 檢測字串，優點為巨量 Token 節省與更快回應。
- `vision_service._create_analysis_response` 使用簡易 nutrition DB（字串比對）做示範分析，並回傳 `AnalyzeFoodResponse`。

## 六、部署與本地執行（快速上手）
1. 進入專案根目錄：

```powershell
cd "C:\Users\ASUS\Desktop\人工智慧與深度學習應用期末專題"
```

2. 啟動全部服務（BFF + 前端）：

```powershell
cmd /c start-all.bat
```

3. 檢查 BFF 健康：

```powershell
Invoke-RestMethod -Uri http://127.0.0.1:8080/health
```

4. 若需單獨啟動 BFF：

```powershell
cd foodlens-web\bff-fastapi
.\.venv\Scripts\python.exe -m uvicorn app.main:app --host 0.0.0.0 --port 8080
```

## 七、測試與驗證
- 已測試 `start-all.bat`：當 8080/8000 被佔用時會嘗試 `taskkill` 釋放，並啟動相應服務；BFF `/health` 回傳 `{"status":"ok"}`。
- 已用 mock provider 驗證 `analyze-food` 與 `analyze-text` 的回傳格式正確性。

## 八、已知問題與解法
- Windows batch 的語法不支援 `else if`，已以巢狀 `if` 修正。
- 日誌檔編碼可能導致中文亂碼，已在 `start-all.bat` 設定 `chcp 65001`。
- 若要整合真實 AI/視覺 API，請在 `bff-fastapi` 內設定環境變數（例如 `VISION_PROVIDER`、API_KEY）並在 `vision_service` 中實作對應呼叫。

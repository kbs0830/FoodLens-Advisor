# FoodLens-Advisor — 完整工作流程

本文檔說明從「取得專案」到「本地測試、提交、推上遠端、部署」的完整步驟，包含常見偵錯與建議。請依環境（Windows）與專案目錄執行。

---

## 1. 取得專案
- 從遠端 clone 或複製整個專案資料夾到本機：

```powershell
# 使用 HTTPS
git clone https://github.com/username/repo.git
# 或 SSH
git clone git@github.com:username/repo.git
```

- 或直接將工作資料夾放到目標路徑（例如桌面）。

## 2. 建立與準備環境（BFF）
- 進入 BFF 資料夾並建立虛擬環境：

```powershell
cd foodlens-web\bff-fastapi
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt
```

- 若系統已有 Python，請確認版本為 3.10/3.11 或相容版本。

## 3. 準備前端（Web client）
- 前端為靜態檔案，放在 `foodlens-web/web-client`。
- 若需要安裝前端套件（非本專案必要），請參考該資料夾 README。

## 4. 一鍵啟動（開發用）
- 專案根目錄提供 `start-all.bat`：會啟動 BFF（uvicorn）與前端（Python http.server），並嘗試釋放 8080/8000 被佔用的 PID。

```powershell
cd "C:\path\to\repo"
cmd /c start-all.bat
```

- 若想單獨啟動 BFF：

```powershell
cd foodlens-web\bff-fastapi
.\.venv\Scripts\python.exe -m uvicorn app.main:app --host 0.0.0.0 --port 8080
```

## 5. 功能測試（API 範例）
- health 檢查：

```powershell
Invoke-RestMethod -Uri http://127.0.0.1:8080/health
# -> {"status":"ok"}
```

- 分析文字（YOLO 結果）範例：

```powershell
$body = @{
  food_items = @("chicken breast","broccoli")
  description = "grilled chicken breast with steamed broccoli"
} | ConvertTo-Json
Invoke-RestMethod -Uri http://127.0.0.1:8080/api/v1/analyze-text -Method POST -Body $body -ContentType 'application/json'
```

## 6. 日誌與偵錯
- 日誌位置：`logs/bff.log`, `logs/web.log`, `logs/bff_stop.log`, `logs/web_stop.log`。
- 常見錯誤與處理：
  - 埠被占用（WinError 10048）：使用 `netstat -aon | findstr ":8080"` 找出 PID，使用 `taskkill /PID <pid> /F` 終止，或改用其它埠。
  - Python 可執行檔找不到：檢查 `.venv\Scripts\python.exe` 是否存在或將 Python 加入 PATH。
  - Batch 語法錯誤：Windows batch 不支援 `else if`，請使用巢狀 `if`。

## 7. 開發工作流程（Branch、Commit、PR）
- 建議的工作分支策略：
  - `main` / `master`：發布用、穩定版。
  - `dev` / `web-dev`：日常開發與整合。
  - feature 分支：`feature/xxx`。

- 範例提交流程：

```powershell
git checkout -b feature/add-vision-provider
# 編輯、測試
git add .
git commit -m "feat: add vision provider integration"
git push origin feature/add-vision-provider
# 到 GitHub 建 PR -> 合併到 dev 或 main
```

## 8. 推上遠端（常見步驟）
- 將本地 commit 推到遠端：

```powershell
# 新增遠端（僅第一次）
git remote add origin https://github.com/username/repo.git
# 推上特定分支
git push -u origin web-dev
```

- 若遠端已有歷史且產生衝突：請使用 `git pull --rebase` 或與團隊協調解決衝突，避免使用 `--force` 推送至共享分支。

## 9. 部署（簡易建議）
- 測試環境：可在 VM 或雲端建立 Python 環境，建立 `.env` 儲存機密（API Key、VISION_PROVIDER），使用 `uvicorn` + process manager（systemd / PM2 / Windows service）來運行 BFF。
- 上線建議：將前端靜態檔部署到 CDN 或靜態主機（GitHub Pages、Netlify），後端部署到 VM / Docker / Cloud Run，並啟用 HTTPS 與監控。

## 10. 自動化測試與 CI（建議）
- 建議使用 GitHub Actions 建立以下 workflow：
  - install deps, run linters
  - run pytest（BFF 單元與整合測試）
  - on push to main -> deploy step

## 11. 安全性與金鑰管理
- 不要把 API Key 寫入程式庫；使用 `.env` 或 CI secret 管理。
- 若使用雲端 AI 服務，限制金鑰可用權限與網域。

## 12. 建議的工作節奏（小組協作）
- 每個 feature branch 小而專，提交訊息要清楚。
- PR 需包含變更說明、測試步驟與預期行為。
- 定期合併 dev -> main，並在 main 上做 release tag。

## 13. 回滾與備援
- 若上線後發現問題，立即在 GitHub 建 issue 並使用 `git revert` 或回滾到先前的 tag，避免直接重寫歷史（force push）。
